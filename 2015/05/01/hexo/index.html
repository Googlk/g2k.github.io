Hexo的搭建教程-1

  • hexo

前言:

Hexo的搭建教程已经不是第一篇了,我在这几天的折腾中大概已经看了不少于20篇,为什么还要重新写一篇教程出来呢?因为她们写的在我看来都不够完美(详细),其实只因为我是小白而已,所以我在这里再总结一遍我在搭建Hexo中走过的路,面向对象为只会用鼠标的孩子,希望这部分人看了这篇教程后也能用上Hexo.

  • 注意:此教程在windows上实测,Mac.类UNIX.Linux.用户请慎重参考.
    好了,我的Hexo装big之路现在开始…
  • 安装Node
    到 Node.js 官网下载相应平台(这里应该是windows平台)的 最新版本 ，一路安装即可。进到官网 你会看到install字样的按钮.点击下载安装就行了.
    • 安装Git
    • 安装sublime(这个不是一定要安装的,只是用来打开.md等文件的工具,如果你有给好的工具请忽略3)
    • 登陆GitHub账号
    • 配置SSH keys
      我们为什么能使本地git项目与远端的github链接起来，就是因为SSH keys的存在。下面我们配置SSH keys。
      首先我们需要检查你电脑上现有的ssh key：
$ cd ~/.ssh 检查本机的ssh密钥，（代码在git上运行）

如果提示：No such file or directory 说明你是第一次使用git。

生成密钥：

ssh-keygen -t rsa -C "123@qq.com"
//注意代码中的C要大写

Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<这里什么也不用填，回车就好>
然后系统会要你输入密码：
Enter passphrase (empty for no passphrase):<输入加密串>
Enter same passphrase again:<再次输入加密串>
在回车中会提示你输入一个密码，这个密码会在你提交项目时使用，如果为空的话提交项目时则不用输入。这个设置是防止别人往你的项目里提交内容。

注意：输入密码的时候没有*字样的，你直接输入就可以了。
添加SSH-Key(先在你github主页右上角找到setting点进去,再在左边找ssh-key)
.ssh 目录下生成两个文件 id_rsa 和 id_rsa.pub
用文本编辑器(这里可用sublime)打开 id_rsa.pub 文件，拷贝其中的内容，将其添加到 Add SSH Key(title那栏可不填,拷内容进key哪里就好)

最后可以验证一下：

ssh -T git@github.com

屏幕显示：
The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
Are you sure you want to continue connecting (yes/no)?

输入yes 回车
Hi *! You’ve successfully authenticated, but GitHub does not provide shell access.

  • 设置用户信息
    首先设置你的用户名密码：
 git config --global user.email "123@qq.com"
 git config --global user.name "123"
//这两行代码在git上打

* 执行如下命令安装hexo：

npm install -g hexo

这里有warn可以不用管。

初始化

然后，执行init命令初始化hexo到你指定的目录（如D://hexo）：

hexo init  //在D://hexo下打开git

生成静态页面

执行如下命令，生成静态页面至 hexo\public\ 目录。

hexo generate

本地启动

执行如下命令，启动本地服务，进行文章预览调试。

hexo server

浏览器输入 http://localhost:4000就可以看到效果。

$ hexo s

*主题
修改主题后没有显示修改的结果，建议执行如下命令，再看看，可以解决很多问题。
$ hexo clean
$ hexo g
$ hexo s
复制主题：

$ git clone https://github.com/litten/hexo-theme-yilia.git themes/yilia
*配置

修改hexo根目录下的 _config.yml ： theme: yilia
注意：Hexo有两个config.yml文件，一个在根目录，一个在theme下，此时修改的是在根目录下的。
一个是对整站的配置H:\hexo_config.yml，另一个是对主题的配置H:\hexo\themes\light_config.yml

    写文章

执行new命令，生成指定名称的文章至hexo\source\_posts\postName.md。

hexo new  "postName" //新建文章,postname就是你的文件名

自定义页面

执行new page命令

hexo new page "about"

在 hexo\source\ 下会生成 about 目录，里面有个index.md，直接编辑就可以了，然后在主题的 _config.yml 中将其配置显示出来。

上述步骤，也可以手工生成，在 hexo\source\ 下手工新建 about 和 index.md 也是完全等价的。

因为markdown对table的支持不好，我是在about中直接建立index.html，里面书写页面内容，hexo会帮你加上头和尾。
设置About问题:在根目录的source下创建About/index.md,还要在主题下的config文件中menu下About中补充: /About 这样就可以了.
